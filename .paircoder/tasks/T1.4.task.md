# Objective

Implement the replay engine in `cli_replay/player.py`. Reads `.clirec` files and writes events to stdout with timing delays, supporting speed control, delay capping, input filtering, and instant dump.

# Implementation Plan

1. Implement `_compute_delay(current_t, prev_t, speed, max_delay, instant) -> float` — pure function
2. Implement `_should_skip(event_type, no_input) -> bool` — pure function
3. Implement `play()` orchestrator:
   - Open file, call `session.read_header()` (header used for future features, ignored in v1)
   - Iterate with `session.iter_events()`
   - For each event: compute delay, sleep, write `data` to `sys.stdout`, flush
   - Handle `--no-input` by skipping `i` events
   - Handle `--instant` by setting all delays to 0

# Acceptance Criteria

- [x] `_compute_delay` returns `(current_t - prev_t) / speed`
- [x] `_compute_delay` caps result at `max_delay`
- [x] `_compute_delay` returns 0.0 when `instant=True`
- [x] `_compute_delay` returns 0.0 for first event (prev_t = 0)
- [x] `_should_skip` returns True for `i` events when `no_input=True`
- [x] `_should_skip` returns False for `o` events regardless
- [x] `play()` writes event data to stdout in order
- [x] `play()` with `--no-input` skips input events
- [x] `play()` with `--instant` never calls `time.sleep`
- [x] `play()` raises `FileNotFoundError` for missing files
- [x] All tests pass with `pytest tests/test_player.py`
- [x] `bpsai-pair arch check cli_replay/player.py` passes

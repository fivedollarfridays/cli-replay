# Objective

Implement read functions in `cli_replay/session.py` and create test fixture files. Completes the session module so both recorder and player can use it.

# Implementation Plan

1. Implement `read_header(f)` — read first line, parse JSON, validate, return dict
2. Implement `iter_events(f)` — lazy iterator yielding validated `SessionEvent` dicts
3. Create `tests/fixtures/sample.clirec` — header + 5 events (input/output/ANSI)
4. Create `tests/fixtures/empty_session.clirec` — header only
5. Create `tests/fixtures/bad_version.clirec` — version: 99 for error testing
6. Write round-trip tests (write → read produces identical data)

# Acceptance Criteria

- [x] `read_header` parses first JSON line and validates it
- [x] `read_header` raises `ValueError` on invalid/missing header or empty file
- [x] `iter_events` yields events lazily (generator, not list)
- [x] `iter_events` validates each event and raises on bad data
- [x] Round-trip: write header+events → seek(0) → read back produces identical data
- [x] `sample.clirec` fixture has header + mix of i/o events with timing
- [x] `empty_session.clirec` has header only, `iter_events` yields nothing
- [x] `bad_version.clirec` causes `read_header` to raise `ValueError`
- [x] All tests pass with `pytest tests/test_session.py`

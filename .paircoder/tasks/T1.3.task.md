# Objective

Implement the PTY-based terminal recording engine in `cli_replay/recorder.py`. This is the highest-complexity task — spawns a shell via PTY, multiplexes I/O with `select`, and logs events to `.clirec` format.

# Implementation Plan

1. Implement `_generate_filename(output: str | None) -> str` — timestamp-based default, or custom name with `.clirec` appended
2. Implement `_build_header() -> dict` — terminal size via `os.get_terminal_size()`, ISO timestamp
3. Implement `record()` orchestrator:
   - Open PTY with `pty.openpty()`
   - Set terminal size on PTY slave via `fcntl`/`struct`
   - Spawn shell subprocess with PTY as stdin/stdout/stderr
   - Save and set `tty.setraw(sys.stdin)` for raw input forwarding
   - `select.select()` loop: read from stdin → write to PTY + log `i`; read from PTY → write to stdout + log `o`
   - Restore terminal settings on exit (try/finally)
   - Print summary to stderr (event count, duration, file path, size)
